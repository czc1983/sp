# Mode2 前端事故复盘与防再犯规则

日期：2026-07-23

## 这次发生了什么

Mode2 页面在修复助手、路线、白膜、分轨等功能时，`web_ui/story_generate_dashboard.html` 这个大体量单文件页面出现了编码/脚本损坏。页面虽然后来用旧备份救回来了，但也把一部分较新的 UI 设计一起回退掉了，所以用户看到的界面变丑、流程状态也有回退感。

这不是普通小 bug，而是工程流程事故：页面能跑回来，不等于产品体验恢复到了事故前。

## 直接原因

1. 对大型 HTML 文件做了高风险整体改写，触发了中文乱码、脚本断裂或结构不完整。
2. 修改前没有建立足够近的 Git checkpoint，导致“恢复到最近稳定 UI”这件事没有清晰落点。
3. 备份文件只证明“能运行”，不证明“视觉和交互是最新版本”。
4. 修复时优先救可运行性，牺牲了新 UI 的状态和体验。
5. 没有把每次关键 UI 改动及时提交到 GitHub，远端仓库不能作为可靠回滚点。

## 根因

Mode2 页面承担了太多职责：资产库、助手、白膜路线、Seedance 路线、服务器 Scail2 路线、分轨、锚点、设置、日志都塞在一个页面里。单文件越大，任何编码错误、批量替换、全文件写入都会放大成灾难。

以后不能再把“大页面能打开”当成验收标准。Mode2 的验收必须同时包括：

1. 页面能启动。
2. 核心按钮存在。
3. 路线状态正确。
4. 助手下一步能判断当前流程。
5. 视觉布局没有明显回退。
6. Git 有可回滚的稳定点。

## 以后必须遵守的规则

### 修改前

1. 先执行 `git status --short`，确认当前脏文件。
2. 对高风险文件先做本地备份，尤其是：
   - `web_ui/story_generate_dashboard.html`
   - `web_ui/splitter_dashboard.html`
   - `web_ui/server.py`
   - `spvideo/scail2_client.py`
3. 修改大型 HTML 时禁止用 PowerShell `Set-Content`、重定向、字符串拼接等方式整文件重写。
4. 优先用 `apply_patch` 做小范围补丁。
5. 如果必须大改 UI，先创建 checkpoint commit 或至少导出 patch。

### 修改中

1. 不在同一个大 HTML 文件里同时改视觉、流程、接口、状态机。
2. 一轮只解决一个明确问题，例如“路线状态栏”或“助手按钮收纳”。
3. 不把实验按钮长期留在生产页面。
4. 不把用户已经验证无效的实验路线继续放在主流程里。
5. UI 逻辑要围绕用户的真实路线：
   - 默认路线：服务器白膜 / Scail2 换人
   - 次选路线：Seedance 直出
   - Seedance 路线不需要分轨、锚点、白膜
   - Scail2 路线才需要分轨、锚点、目标人物确认

### 修改后

1. 对 HTML 提取 `<script>` 后执行 `node --check`。
2. 执行 `git diff --check`。
3. 检查文件里是否出现 Unicode replacement character 乱码字符。
4. 重启前后端并确认页面可打开。
5. 有 UI 改动时必须截图留档，至少保存一张“修改后可用状态”。
6. 用户确认稳定后，立即提交 Git；需要同步远端时再 push。

## GitHub 智能处理原则

GitHub 不能等项目坏了才想起来用。以后按“工作前 checkpoint，稳定后 commit，阶段性 push”的节奏走。

推荐节奏：

1. 开工前：运行 `scripts/safe_checkpoint.ps1 -Name "before-mode2-ui-change"`。
2. 大改前：先 commit 当前可运行版本。
3. 每完成一个可独立验证的小功能：commit 一次。
4. 用户确认某个版本好用：立刻 commit，并询问是否 push 到 GitHub。
5. 遇到页面损坏：优先回到最近一个明确 commit，不从随机 `.bak` 里盲恢复。

不默认自动 push 的原因：远端 GitHub 是共享回滚点，不能把半坏版本推上去污染远端。除非用户明确说“提交并推送”，否则只做本地 checkpoint 和建议。

## 恢复时的正确顺序

1. 停止继续堆功能。
2. 记录当前坏文件为 `.corrupt_时间.bak`。
3. 从最近明确 checkpoint 或用户认可的备份恢复。
4. 先跑语法检查。
5. 再恢复必要的新功能。
6. 最后单独补 UI，不把 UI 重做和功能修复混在一起。

## 后续重构方向

1. 把 `story_generate_dashboard.html` 拆成更小的模块，至少拆出：
   - 助手面板
   - 路线状态栏
   - 资产卡片
   - 生成区媒体预览
   - 设置弹窗
2. 页面主流程只保留必要按钮：
   - 带我做视频
   - 当前路线切换
   - 设置
   - 当前步骤的唯一主操作
3. 复杂解释放到助手问答，不占页面固定空间。
4. 所有运行中/推理中任务统一显示脉冲状态。
5. 生成区显示缩略图/视频预览，不显示大段路径和代码。

## 给以后 Codex/子智能体的硬规则

1. 不要在没有 checkpoint 的情况下大改 Mode2 UI。
2. 不要用全文件写入方式改含中文的大 HTML。
3. 不要恢复旧备份后直接宣称完成，必须说明 UI 是否回退。
4. 不要让多个智能体同时写同一个大 HTML 文件。
5. 不要把实验路线混进用户主流程。
6. 做完任何关键流程改动，必须给出验证结果和 Git 状态。
