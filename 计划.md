# 短剧转绘海外 — 批量自动化项目计划

## 一、项目目标

国内短剧 → AI 转绘 + 本地化 → 目标市场短剧（西语/英语等）

核心原则：
- 资产送审不等人，分镜同步进行
- 过审率低于 50% 会被拒收，内部质检必须前置
- 100% 移除中国元素

---

## 二、API 供应商

| 供应商 | 用途 | 说明 |
|--------|------|------|
| **硅基流动** | LLM 翻译/字幕、TTS 配音、语音识别 | DeepSeek-V3、CosyVoice2、SenseVoice |
| **阿里百炼** | 图像生成（资产/海报）、动作迁移（Wan2.2-Animate）、视觉质检 | Qwen-Image、Wan2.2-Animate、Qwen-VL-Max |
| **本地 3060 Ti 8G** | 镜头切割、姿态提取、人物抠图、口型同步、FFmpeg | PySceneDetect + YOLO + DWPose + RVM + Wav2Lip |

> 注：Wan2.2-Animate 动作迁移是阿里百炼 API（云端），本地 3060 跑不动，但 DWPose / RVM / Wav2Lip 本地跑足够。

## 2026-07-17 工作记录：路线分叉与转绘问题结论

### 1. 当前转绘链路的关键结论

今天确认了一件容易误判的事：`person_count = -1`、`segment_type = without_human` 不会直接阻止手动转绘。

手动点“转绘”时，后端真正提交给云端的是：
- 当前片段视频 `video_path`
- 导演计划里的角色列表
- 角色参考图
- 手工标注/素材里找到的角色位置 `source_x`
- 选择的后端，如 SCAIL-2、SCAIL-2 Colored Mask、Bernini、RunningHub Bernini、Wan2.2

也就是说，13 号片段即使在切片结果里被误判成 `p-1 / without_human`，只要手工配置了导演角色并点击转绘，它仍然会上传视频、参考图和相关参数去跑云端工作流。它不是“没跑”的直接原因。

但这个误判会间接影响：
- 自动导演可能不会主动问“这段里的人是谁”
- 以后批量流程可能跳过这段
- 没有手工导演标点时，角色位置 fallback 会变差
- UI 会误导排查方向

所以 13 号“跑了但没换人”的主要排查方向不是 `person_count`，而是：参考图是否被模型吃进去、参考图和蒙版是否强绑定、多人颜色顺序是否对应、云端工作流是否只是把蒙版当定位而没有强制身份替换。

### 2. 这几天 SCAIL2 / Bernini 试验的状态

- 24 号三人片段：曾经有一版两边人物能换，中间人物换不掉；暂时放下三人问题。
- 56 号两人片段：理论上更简单，但仍然只换一个人或输出异常，说明问题不是单纯“三人太难”。
- 13 号两人片段：SAM3/彩色蒙版看起来能生成，但转绘视频仍然像没吃参考图。
- 普通 SCAIL-2 ComfyUI 路径主要使用 `SCAIL2SimpleVideo + SCAIL2ReferencePack + SCAIL2ReferenceSAMBuilder`，后台常看到白色单人 mask，不等于最终显式使用彩色 mask。
- 已新增独立测试后端 `SCAIL-2 Colored Mask`，走显式 `SCAIL2ColoredMask + WanSCAILToVideo`，并加了 `AUTO_SEG_V2` 自动分段日志标记。
- 动态参考图测试结果不理想：参考图可以技术上塞成视频 batch，但 Wan/SCAIL 不像真正逐帧吃动态参考，更像吃静态身份/CLIP 参考，容易贴出重复人物或不稳定画面。
- Bernini / RunningHub 工作流已能提交，但遇到过模型通道不匹配、任务失败、输出不按参考图等问题，说明还没到稳定生产状态。

### 3. 新路线：模式 2「分镜重生」

用户现在倾向换路线：不再围绕“原视频转绘换人”死磕，而是尝试：

原视频 -> 反推故事 -> 逐镜头分镜 -> 分镜提示词 -> 重新生成全新片子

这个路线的目标不是 1:1 还原原视频所有动作，而是尽量保持剧情、人物关系、镜头节奏和情绪走向。优点是可能避开多人换脸/换身/蒙版绑定这些 SCAIL2 难点；缺点是动作、构图、连续性和人物一致性需要新的一套控制方法。

已在 Web UI 顶部新增“模式”入口：
- 模式 1：转绘替换。继续使用当前切片、导演标注、参考图、SCAIL2/Wan2.2/Bernini 流程。
- 模式 2：分镜重生。已独立成 `story_generate_dashboard.html`，不复用旧转绘界面。当前第 4 个功能区已接入 Seedance2 通道A草稿界面，参考 `F:\xin\T8-penguin-canvas` 的视频控件思路，支持 4 个 A-seedance-2.0 模型、比例/时长/分辨率、最多 4 张参考图、3 个参考视频、3 个参考音频，并可生成/复制/保存 payload 草稿。
- 注意：Seedance2 通道A目前只做“任务草稿/手工测试入口”，不直接提交 API、不扣费；等新路线确认后，再单独接正式提交后端，保持新模式和旧模式解耦。

2026-07-17 已补资产绑定层：
- 新模式的分镜草案接口 `/api/storyboard-draft` 会生成角色、场景、道具资产，并给每个资产写入 `used_shots` 和 `usage_count`。
- 页面 02「资产绑定」会显示类似“用于 5 个分镜”的资产使用范围；编辑资产时也能看到具体分镜编号，如 `S001、S002`。

- 这层逻辑的意义：换一张角色图或场景图时，可以知道会影响哪些分镜，不再只是“图片列表”；后面批量生成、重跑局部镜头、质检问题定位都要依赖这个索引。

2026-07-17 已补分镜提示词编译器：

## 五、下一步执行顺序

### 2026-07-18：Mode2 分镜重生当前优先路线

当前 Mode2 已经独立出“分镜草案 → 资产绑定 → 资产提纯 → Seedance 通道A生成”的新路线。旧模式1只作为参考代码，不直接复用旧转绘链路。

#### P0：人物提纯升级：接 SAM3 人物轨迹

目标：人物资产不能再只靠人脸 crop 或同一批关键帧猜身份，要按时间和人物身份分轨。

实现口径：
- 使用 Mode2 自己的入口，不直接调用模式1转绘流程。
- 输入：原视频、Mode2 全片理解出的角色列表、角色出现时间段、关键帧候选。
- 对每个角色在其出现时间窗内找代表帧/代表点，生成 SAM3 人物轨迹和 mask。
- 将轨迹结果写入 Mode2 资产字段：`refined_source_image`、`refined_mask_image`、`track_path`、`track_status`、`refinement_status`。
- 如果两个角色抽到同一批源帧或 mask 高度重叠，标记为 `needs_manual_review`，不能假装 ready。
- UI 需要能看到：该角色来自哪几个时间段、哪张代表帧、哪条 SAM3 轨迹、是否撞轨。

要解决的问题：
- 两个人同框时，女主和黑衣男子不能共用同一张源图。
- 三人/多人场景里，不能只按左右顺序硬猜人物身份。
- 后续 Seedance 生成人物参考时，角色图必须能追溯到对应人物轨迹。

#### P1：物品/场景多帧择优

目标：同一个资产不能只跑一张候选图。要从多个候选帧里跑检测，自动选 mask 最干净、最可信的一张。

实现口径：
- 场景资产：对多个候选帧跑 `GroundingDINO + SAM` 找人物 mask，再用 `big-lama.pt` 做 LaMa 去人补背景。
- 物品资产：对多个候选帧跑 `GroundingDINO + SAM`，按资产英文目标词检测并裁剪。
- 每张候选帧都记录质量分：mask 面积、bbox 面积、碎片数、人物占比、边缘清晰度。
- 自动选择质量最高的结果写入 `refined_source_image`，其余写入 `refined_source_images` 或 evidence。
- 如果 mask 太大、太散、几乎为空，标记 `needs_manual_review`。

要解决的问题：
- “粗链条/粗铁链”这种同类物很多时，不能把整面墙和人物一起当成物品。
- “床”这种语义容易错配的物品，必须以画面检测结果为准，语义只是线索。
- 场景源图如果人物占比太高，要优先变成“镜头参考”，不要冒充干净背景。

#### P2：UI 增强：单资产重新提纯 + 人工确认

目标：资产页不仅能一键提纯，还要能逐个资产核对和重跑。

实现口径：
- 每个资产卡片增加“重新提纯”按钮。
- 支持只重跑当前资产，不影响其他资产。
- 弹窗里同时显示：原证据图、mask、提纯图、目标替换图、质量判断说明。
- 支持人工确认：`确认可用`、`需要换图`、`只作镜头参考`、`忽略此资产`。
- 用户人工确认结果写回 Mode2 资产表，供 Seedance 生成策略读取。

要解决的问题：
- 自动模型一定会有误检，必须给人工快速纠偏入口。
- 资产很多时，不能靠大弹窗一张张放大看，要能快速扫、快速确认。
- 提纯失败的资产不能阻塞整条分镜流程。

#### P3：Seedance 生成接入资产策略

目标：Seedance 生成时不再把所有图一股脑塞进去，而是根据资产类型选择正确用途。

实现口径：
- 人物：优先使用人工替换图/角色图；若没有，则用 `refined_source_image` 作为身份候选参考。
- 场景：如果有 `clean_background`，作为背景/环境参考；如果只是 `shot_reference`，只作为镜头构图/动作关系参考。
- 物品：优先使用 `prop_cutout`；若质量为 `needs_manual_review`，提示人工确认后再用。
- 原视频继续作为动作骨架参考：人物站位、动作、表情、遮挡、镜头运动以原视频为主。
- 提示词中明确写资产绑定关系，避免人物互换、混脸、串衣服。

要解决的问题：
- 生成模型不能重新理解剧情，必须按原视频动作骨架走。
- 不同资产类型要分用途：人物锁身份，场景锁环境，物品锁道具，原视频锁动作。
- 后续批量生成时，每个镜头自动带入本镜头真正相关的资产。

## 2026-07-22 工作记录：白膜/3D 控制路线与前端修复

### 1. 当前路线判断

Seedance2.0 能吃“视觉控制视频”，但对抽象 3D 代理理解很差。简单低模、箭头、线条、数据蓝图都不稳定，容易丢镜头、改动作或把人物关系看错。

目前更有价值的发现是：
- 参考图权限很高，可以先让模型把原视频人物替换成目标参考人物，再转成白膜/陶瓷/兵马俑等去身份控制视频。
- 这种“先替换身份，再去身份化”的控制视频污染更少，因为白膜里已经带了目标人物的动作关系和身份布局。
- 不要在提示词里讲剧情，剧情描述会让模型自由生成；提示词只写“替换人物、保留动作、完整分镜、控制视频用途”。

### 2. 白膜/3D 实验结论

2.5D 白膜的问题：
- 深度/方向不够强时，Seedance 会把脸朝向、遮挡、人物距离理解反。
- 箭头、明显线条、彩色骨架会污染生成画面，不适合作为正式控制视频。
- 表情可以用极淡脸部阴影/五官凹凸，但不能画成标记线。

3D 控制的问题：
- Seedance 不理解过于抽象的 3D 代理，只吃“像电影画面一样”的视觉。
- 只有当 3D 做到接近原视频的空间、人物、动作、接触、表情和分镜，才可能成为有效控制视频。
- 对亲吻、贴脸、拥抱等复杂接触镜头，简单 3D 代理不够。

### 6. 下一步

优先不要继续做抽象 3D。下一步应做“去身份化但像真实视频”的控制视频：
- 先用参考图替换原片人物。
- 再把替换后的视频转成陶瓷/兵马俑/白膜风格控制视频。
- 再把目标参考图 + 控制视频送 Seedance，让它转回自然写实。

目标是减少原视频人物污染，同时保留完整分镜、动作、接触关系、人物数量和镜头时长。
